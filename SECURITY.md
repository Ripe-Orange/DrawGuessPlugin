# 安全政策

## 报告安全漏洞

我们非常重视项目的安全性。如果您发现了任何安全漏洞，请通过以下方式报告：

### 联系方式

- **GitHub Issues**：使用`security`标签创建 Issue

### 报告内容

在报告安全漏洞时，请提供以下信息：

- 漏洞的详细描述
- 复现步骤
- 受影响的版本
- 可能的影响范围
- 任何已知的缓解措施

我们将在收到报告后及时回复您，并与您合作解决问题。

## 安全更新流程

1. 收到安全漏洞报告
2. 确认漏洞并评估其严重程度
3. 开发修复方案
4. 测试修复方案
5. 发布包含修复的新版本
6. 更新安全公告

## 安全版本支持

我们提供对以下版本的安全更新：

- 最新的稳定版本
- 最新的预发布版本

## 安全最佳实践

在使用本项目时，请遵循以下安全最佳实践：

1. 始终使用最新版本的插件
2. 仅从官方渠道下载插件
3. 定期检查安全公告
4. 在使用第三方插件时保持警惕
5. 定期备份您的数据

## 安全公告

所有安全公告将发布在：

- GitHub Releases
- 项目 Discussions

本安全政策受项目许可证（MIT）保护。
